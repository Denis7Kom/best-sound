# Security Policy

## Supported project

This repository contains the public static website files for BEST SOUND.

The currently supported branch is:

- `main`

Security reports should refer to the current state of `main` unless another branch is explicitly requested.

## Reporting a vulnerability

If you find a security issue, do not disclose it publicly before it has been reviewed.

Please report the issue privately to the repository owner through GitHub or another private contact channel available to you.

When reporting, include:

- a clear description of the issue;
- affected file or URL;
- steps to reproduce;
- potential impact;
- screenshots or proof of concept if relevant;
- suggested fix if available.

## Responsible disclosure rules

Do not:

- exploit the issue beyond what is necessary to prove it exists;
- attack visitors, users, third-party services, or infrastructure;
- access, alter, destroy, or exfiltrate data;
- run automated destructive scans;
- publicly disclose the issue before it has been reviewed.

## Scope

In scope:

- HTML, CSS, SVG, JavaScript and static assets in this repository;
- public GitHub Pages deployment of this repository;
- obvious misconfigurations in static files, links, forms, metadata or privacy/security headers where applicable.

Out of scope:

- attacks against GitHub itself;
- attacks against third-party services not controlled by this repository;
- spam, social engineering, phishing or physical attacks;
- denial-of-service testing;
- reports based only on missing backend protections when no backend exists.

## No permission to copy or reuse

This security policy does not grant any license to use, copy, clone, redistribute, resell, rebrand, deploy, or publish this website or any part of its source code, design, text, layout, assets, or structure.

All repository content remains subject to the repository license.

## Expected response

The maintainer will review valid reports as soon as reasonably possible and may request more information.

Accepted reports may be fixed directly in `main` or through a pull request, depending on urgency.
